## 用量分档：你属于哪一种

### 轻度用户（每月跑 10-20 个任务）

典型场景：偶尔写个脚本、处理几次数据、改几个 Bug。

- **官方 Pro**：¥1450/月，大部分额度浪费。不划算。
- **API 中转**：按实际 token 消耗，可能只花 ¥30-100。划算。
- **镜像站**：入门套餐 ¥几十。最便宜，但功能可能有限制。

**结论：轻度用户选 API 中转或镜像站，别订 Pro。**

### 中度用户（每月跑 50-100 个任务）

典型场景：每天都在用 Codex 写代码、改项目、跑测试。

- **官方 Pro**：¥1450/月，额度能用上，但用完了还是要加。
- **API 中转**：按量计费可能花 ¥200-500。如果任务重的月份会更多。
- **镜像站**：中档套餐可能不够用，要叠加购买。

**结论：中度用户 API 中转性价比最高。Pro 适合不想算账、每月稳定重度使用的人。**

### 重度用户（每月 200+ 任务，长任务为主）

典型场景：全职用 Codex 干活，8 小时级别的长任务常跑。

- **官方 Pro**：¥1450/月封顶（在额度内），可预测。
- **API 中转**：长任务 token 消耗大，可能花 ¥1000+。不一定比 Pro 便宜。
- **镜像站**：通常不适合重度长任务场景。

**结论：重度用户直接上 Pro，月费可控。**

## 隐性成本别忘了

### 官方 Pro 的隐性成本

- 需要海外信用卡或协助充值（一次性手续费）
- 需要稳定的网络环境
- 超出额度后等待补充或降级体验

### API 中转的隐性成本

- 需要懂怎么调 API（有开发门槛）
- Token 消耗不透明的任务，可能账单超预期
- 需要自己管理 prompt 效率

### 镜像站的隐性成本

- 功能可能不完整（平台取决于实际接入的模型和权限）
- 速度取决于第三方服务器和网络
- 平台稳定性不如官方直连
