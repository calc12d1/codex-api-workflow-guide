## Codex 和普通 ChatGPT 对话的本质区别

一句话：ChatGPT 给你文字，Codex 给你结果。

| | ChatGPT 对话 | Codex |
|---|---|---|
| 你说"帮我整理这个表格" | 给你一段教程文字 | 直接处理表格，输出结果文件 |
| 你说"分析这份数据" | 给你分析思路 | 跑代码，出图表 |
| 你说"把这 50 份合同里的关键条款提取出来" | 告诉你怎么做 | 执行提取，输出汇总 |

**对非程序员来说，区别就是：一个教你怎么做，一个直接帮你做完。**

## 非程序员的 5 种用法

### 1. 表格和数据处理

给 Codex 一个 Excel 或 CSV，用自然语言描述要什么结果。

例子："把这份销售数据按地区分组，算每个地区的月均增长率，输出一张图。"

Codex 会写脚本 → 跑数据 → 出图表 → 你下载结果。你全程不需要看代码。

### 2. 文档批量处理

有 100 份合同要提取关键日期？50 份简历要筛选条件？

例子："从这些 PDF 里提取所有含'违约金'的条款，按金额从高到低排列。"

法务团队就是这么用的。OpenAI 内部数据显示法务的 token 消耗量 6 个月涨了 13 倍。

### 3. 自动化重复工作

每天要手动做的事：下载报表 → 格式转换 → 发邮件 → 更新看板。

给 Codex 描述这个流程，它帮你写自动化脚本。下次一键跑完。

### 4. 快速做原型

想要一个简单的内部工具（比如报销审批表、客户信息查询页面），不用排开发队列。

描述你要什么，Codex 直接生成可用的工具。不完美，但能先跑起来。

### 5. 数据可视化

把枯燥的数字变成能给老板看的图表。

"用这份季度数据做一个柱状图+折线图的组合图，风格参考 McKinsey 报告。"

## 怎么跟 Codex 说话才有效

非程序员最容易犯的错：**描述太模糊。**

| 效果差 | 效果好 |
|---|---|
| "帮我处理这个数据" | "把 A 列按日期排序，删掉 B 列为空的行，输出到新文件" |
| "做个图" | "用柱状图展示各月销售额，横轴是月份，纵轴是金额（万元）" |
| "分析一下" | "计算每个产品的环比增长率，标出下降超过 10% 的" |

关键原则：**说清楚输入是什么、输出要什么格式、处理逻辑是什么。** 你不需要懂代码，但需要知道自己要什么。
