# 阿里封杀 Claude Code，国内开发者还能怎么用

更新时间：2026/07/07 · 账号 A · 资讯向

阿里巴巴把 Claude Code 列为高风险软件，从 7 月 10 日起禁止员工使用，改用自研的 Qoder 工具。这事在开发者圈炸开了，但真正值得看的不是"封杀"两个字，而是背后那条越来越清楚的线。

---

## 到底发生了什么

按 Reuters 和 Morningstar 的报道，阿里内部把 Claude Code 归类为高风险软件，理由是安全担忧。

导火索是一份报告：有人发现某个版本的 Claude Code 能悄悄识别中国用户。

Anthropic 那边的回应是——这是今年 3 月上线的一个实验，本意是防止未授权的转售商滥用账号、防止模型被蒸馏。工程师 Thariq Shihipar 在 X 上说，更强的防护措施已经替换了这个实验，"其实我们早就想把它撤掉了"。

时间线摆在这：

- **3 月**：Anthropic 上线识别机制（官方说是防滥用）
- **近期**：机制被曝光，阿里定性为高风险
- **7 月 10 日**：阿里正式禁用，员工转用 Qoder

---

## 这事到底谁有理

两边的说法都有道理，也都有立场。

**Anthropic 的逻辑**：它本来就禁止中国公司用它的模型。识别用户、防止蒸馏，从它的商业和合规角度是自保。蒸馏就是用别的模型的输出来训练自己的模型——Anthropic 一直防这个。

**阿里的逻辑**：一个能识别、能追踪用户的境外工具，放在核心开发流程里，本身就是风险。不管对方是不是善意，先撤掉是稳妥的选择。

说白了，这不是谁对谁错的问题，是双方本来就站在对立的位置上。一个要防外部访问，一个要防外部工具。撞上了，很正常。

---

## 对国内开发者的真实影响

如果你是个人开发者，之前在用 Claude Code，现在的情况是：

- **官方直连**：本来就有门槛（需要海外身份、海外支付），现在识别机制更明确，风险更高
- **企业环境**：像阿里这样的大厂会主动禁用，你在公司里可能直接没得用
- **自研替代**：国产工具（Qoder、通义灵码等）在补位，但成熟度和 Claude Code 还有差距

关键判断：Claude 的模型能力仍然是第一梯队，但"能不能稳定用到"这个问题，对国内用户越来越突出。
